# LongWangClaw｜配置与私有化指南

公开仓只保存可复用的工作区资产、模板和占位 profile。所有账号态、token、私钥、真实 OpenClaw/OpenCode/Codex 配置都必须留在用户本机。

## 统一 profile

公开示例：

```text
workspace/config/longwang.example.json
```

本机私有文件：

```text
workspace/config/longwang.local.json
```

`longwang.local.json` 已在 `.gitignore` 中，必须由用户本机填写，不入仓。

## profile 负责什么

`longwang.local.json` 集中管理：

- OpenClaw workspace 路径
- OpenCode / Codex / Python / Node / Docker 等命令名
- 默认模型 provider 与模型名
- OpenClaw `openclaw.json` 的关键字段
- OpenCode `opencode.json` 的 provider 与插件路径
- Codex `config.toml` 的 provider、模型、trusted projects
- Feishu app、wiki、账号占位
- FOFA / Hunter / Shodan key
- local Docker 或远程 Docker/CVM 的 runtime 根目录、公开访问 URL、必要 SSH 字段
- cron 五条 lane 的 schedule、model、session、delivery

`workspace/config/loop9-dispatch.json` 继续只负责 Loop9 调度参数，例如并发、manualQueue、launchGuard、dockerCleanup。不要把它和全局 profile 合并。

## 渲染器

dry-run：

```bash
python3 scripts/render_longwang_config.py --profile workspace/config/longwang.local.json --dry-run
```

写入本机：

```bash
python3 scripts/render_longwang_config.py --profile workspace/config/longwang.local.json --write
```

渲染目标：

- `~/.openclaw/openclaw.json`
- `~/.config/opencode/opencode.json`
- `~/.codex/config.toml`
- `~/.openclaw/cron/jobs.json`
- `~/.openclaw/workspace/config/longwang.env`
- `~/.openclaw/workspace/Super8/.opencode/agents/*.md` 的 `model` / `variant` frontmatter

默认不会写文件，必须显式传 `--write`。

## 模板

模板目录：

```text
templates/openclaw/openclaw.json.tpl
templates/opencode/opencode.json.tpl
templates/codex/config.toml.tpl
templates/cron/jobs.json.tpl
templates/env/longwang.env.tpl
```

OpenCode agent 文件不单独维护一份完整 `.tpl`。渲染器会读取公开仓里的 `workspace/Super8/.opencode/agents/*.md`，只替换 frontmatter 里的 `model` 和 `variant`，再写到用户本机 `paths.super8Root` 下。这样避免复制四份大模板，同时让模型配置仍由 `longwang.local.json` 统一管理。

模板语法：

- `{{raw:path.to.value}}`：原样插入。
- `{{json:path.to.value}}`：JSON 字面量。
- `{{toml:path.to.value}}`：TOML 字面量。
- `{{sh:path.to.value}}`：shell-safe 单值。
- `{{sh:?path.to.optional}}`：shell-safe 可选值；字段缺失时渲染为空字符串，主要用于 local Docker 不需要的 SSH 字段。

## 模型配置边界

`longwang.example.json` 里的模型名只是公开示例 profile，用来说明字段形状。真实使用时应在 `longwang.local.json` 中改成用户本机可用的 provider/model。

渲染器会把 OpenCode agents 的 `model` / `variant` frontmatter 写成 profile 中的值，所以不要直接把公开仓里的示例模型名理解成硬编码能力承诺。

## Runtime 配置

`remoteRuntime.kind` 当前支持两类公开配置口径：

- `local-docker`：本机 Docker。需要 `commands.docker`、`remoteRuntime.runtimeRoot`、`remoteRuntime.dockerRegistryPath`、`remoteRuntime.publicBaseUrl`，不要求 SSH/CVM 字段。
- `tencent-cvm` / `ssh-docker`：远程 Docker/CVM。除 runtimeRoot 等字段外，还需要 `sshHost`、`sshUser`、`sshKeyPath`、`publicBaseUrl`。

`runtimeProfiles` 只是示例矩阵，用户可以把其中一个 profile 的值复制到 `remoteRuntime` 作为当前 active runtime。doctor 以 `remoteRuntime` 为实际检查对象。

## 必须本机填写的私有值

至少需要按实际环境填写：

- `models.providers.openai.apiKey`
- `feishu.appId`
- `feishu.appSecret`
- `feishu.accounts.*`
- `assetSearch.fofaKey`
- `assetSearch.hunterApiKey`
- `assetSearch.shodanKey`
- `remoteRuntime.*` 中当前 active runtime 需要的字段
- `cron.defaultSessionKey`
- `cron.feishuSessionKey`
- `cron.delivery.*`

如果只先跑 Audit / Real-PoC / 本地报告，可以暂时不启用 Feishu、远程 runtime、cron。doctor 会把对应 lane 标成 blocked/partial，而不会影响其它 lane 的判断。

## 资产搜索 key

`workspace/bin/loop9-asset-evidence-ad-hoc-probe.py` 不再内置 FOFA / Hunter / Shodan 默认明文 key。

运行时从环境变量读取：

```bash
export FOFA_KEY=...
export HUNTER_API_KEY=...
export SHODAN_KEY=...
```

或让 `render_longwang_config.py` 把它们写入本机 `~/.openclaw/workspace/config/longwang.env`，再由运行入口 source。

## 质检

```bash
python3 scripts/longwang_doctor.py --profile workspace/config/longwang.local.json
python3 scripts/longwang_doctor.py --profile workspace/config/longwang.local.json --json
```

doctor 的输出不是发布证明，而是给 AI/维护者看的起跑条件检查。它应该明确指出哪条 lane 已经 ready，哪条还缺命令、配置、模板、插件依赖或私有值。

可选合成契约夹具检查：

```bash
python3 scripts/longwang_doctor.py --profile workspace/config/longwang.example.json --contract-fixture fixtures/contract --json
```

这只检查公开仓自带的最小 contract fixture，不读取真实 target，也不触发真实上传。
