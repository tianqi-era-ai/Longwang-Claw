# 龙王小龙虾｜仓库填充计划

这个仓库现在已经不是“继续发散补资产”的状态，而是“公开闭包收口”的状态。
下面只记录当前边界，不再维持滚动式阶段任务。

## 当前结论

`Longwang-Claw` 只保留能直接回填到 `~/.openclaw` 的公开工作区闭包。
它的任务是装配，不是收集运行态数据，也不是收集案例产物。

- `extensions/openclaw-lark/`

- `docs/`

## 明确不纳入

- `reports/`
- `runs/`
- `targets/`
- `logs/`
- `memory/`
- `~/.openclaw/openclaw.json`
- `.config/opencode/`
- `.codex/sessions/`
- `node_modules/`
- `__pycache__/`
- 任何宣发材料、案例报告、演示稿、CI、smoke、target-sync 资产
- 任何本机运行态缓存、会话、临时目录和账号态配置

## 装配方式

唯一推荐入口是：

```bash
python3 scripts/bootstrap_openclaw_layout.py --workspace-root ~/.openclaw/workspace
```

它只复制仓库里已经存在的文件，不做 manifest，不安装运行时，不补幻想目录。

## 装配原则

1. 保持 `workspace/` 的原样路径关系
2. `skills/` 必须连同 `references/`、`scripts/`、`template/` 一起带上
3. 不把运行态目录、靶标源码、报告产物、缓存、账号态配置当作开源资产
4. 如果某个文件只在本机运行态出现，不要把它补成仓库新文件

## 这版仓库的用途

给 AI 或维护者一份能直接读、能直接装配、能直接回填的公开工作区。
不是宣传仓，也不是证据仓。
