# 龙王小龙虾｜公开仓边界说明

这个仓库的当前目标已经从“继续筛更多文件”收口为“端到端可装配的公开工作区”。

## 当前公开形态

`LongWangClaw` 保留 `~/.openclaw/workspace` 里能支撑核心闭环的工作流资产，并额外提供：

- 统一公开 profile：`workspace/config/longwang.example.json`
- 本机私有模板：`templates/`
- 配置渲染器：`scripts/render_longwang_config.py`
- 全局质检：`scripts/longwang_doctor.py`
- 端到端运行手册：`docs/端到端运行手册.md`

## 当前闭环边界

本仓库要支撑的完整链路是：

1. Audit
2. Real-PoC 静态编写
3. 静态 Audit/PoC 产物上传
4. 动态环境搭建与 PoC 复现
5. 本地标准交付报告生成
6. 动态复现后的标准报告上传

两段上传要分开理解：

- `loop9-feishu-publisher`：上传 raw audit / PoC 过程产物。
- `loop9-feishu-delivery-publisher`：上传标准交付报告树。

## 判断方式

仓库完整性不靠“目录看起来很多”，而靠：

```bash
python3 scripts/longwang_doctor.py --profile workspace/config/longwang.local.json --json
```

doctor 应该能按七条 lane 输出起跑状态：

- `audit`
- `real_poc_static`
- `static_feishu_publish`
- `dynamic_verify`
- `delivery_report`
- `delivery_feishu_publish`
- `cron`

示例 profile 没有私有值，所以部分 lane blocked 是正常的。真正的发布前检查应使用用户本机填写后的 `longwang.local.json`。
