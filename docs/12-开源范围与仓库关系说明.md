# 龙王小龙虾｜开源范围与仓库关系说明

这份文件按当前公开仓状态写，不再保留“未来再拍板”的筹备口径。

## 当前仓库定位

`LongWangClaw` 是第三个公开整理仓。它不是把本机 `~/.openclaw` 和 `~/.openclaw/workspace/Super8` 原样打包，也不是把工作流重新改造成 `src/` 工程包。

当前主形态是：

> 可装配回 `~/.openclaw` 的公开 OpenClaw 工作区资产。

也就是说，公开仓保留能让别人装配、配置、质检、继续运行的核心闭包；真实账号态、运行产物、靶标源码、报告和缓存日志不进入仓库。

## 两个来源层

### `~/.openclaw` 控制面 / 宿主层

公开仓从这一层摘取的是可复用工作区资产：

- `workspace/bin/`
- `workspace/lib/`
- `workspace/config/`
- `workspace/hooks/`
- `workspace/plugins/`
- `workspace/skills/`
- `workspace/plans/`
- `workspace/heartbeat/`
- `workspace/prompts/`
- `extensions/openclaw-lark/`

这一层支撑的是 OpenClaw 作为 AI OS / 控制面的角色：入口、skill、插件、长任务、cron、状态和外部通道。

### `~/.openclaw/workspace/Super8` 工作流 / 执行主线层

公开仓从这一层摘取的是 Loop9 可运行主线：

- `workspace/Super8/.opencode/command/`
- `workspace/Super8/.opencode/agents/`
- `workspace/Super8/.opencode/_scripts/`
- `workspace/Super8/.opencode/_xml/`
- `workspace/Super8/工作流_提示词工程/`
- `workspace/Super8/START_HERE_Loop9.md`
- `workspace/Super8/README.loop9-local.md`

这一层支撑的是 Audit、Real-PoC、长任务 observe、wrapper 发车和状态查看。

### `harness/` 进阶工程层

Harness 单独开目录，但不作为新的运行主入口。

公开仓里的 `harness/` 只放 active 控制面纪律：

- parent / child stage 边界
- runtime isolation 规则
- timeout / model wobble 归因纪律
- handoff / verdict / receipt 的薄写回边界
- 防止厚 orchestrator / 厚 queue 重新夺权的约束

## 当前公开闭包

公开仓按端到端六段链路收口：

1. Audit：`loop9-wrapped-audit`
2. Real-PoC 静态编写：`loop9-real-poc`
3. 静态 Audit/PoC 产物上传：`loop9-feishu-publisher`
4. 动态环境搭建与 PoC 复现：`loop9-verify-v4`
5. 本地标准交付报告：`loop9-delivery-reports`
6. 动态复现后的标准报告上传：`loop9-feishu-delivery-publisher`

对应的装配、私有配置渲染和全局质检入口在：

- `scripts/bootstrap_openclaw_layout.py`
- `scripts/render_longwang_config.py`
- `scripts/longwang_doctor.py`
- `workspace/config/longwang.example.json`
- `templates/`

## 为什么不直接全塞

原始计划里“全量公开”的正确落点不是无差别公开本机所有目录，而是公开产品和工作流的可复用闭包。

以下内容不进入公开仓：

- `reports/`
- `runs/`
- `targets/`
- `logs/`
- `memory/`
- `cron/runs/`
- `~/.openclaw/openclaw.json`
- `~/.config/opencode/opencode.json`
- `~/.codex/config.toml`
- `~/.codex/sessions/`
- `.env*`
- 私钥、token、真实 API key
- 第三方靶标源码
- 运行中间产物
- 已交付漏洞报告
- 宣发/运营材料
- 首发案例名单

这些内容要么是运行态，要么是账号态，要么是对外案例/宣发材料，不属于公开可装配工作区资产。

## 公开仓和运行态的关系

公开仓只提供：

- 工作流代码和 skill
- prompt / agent / wrapper
- OpenClaw 飞书插件源码
- profile 示例和模板
- 装配脚本
- readiness doctor
- 端到端运行说明
- Harness 进阶工程说明

用户本机填写 `workspace/config/longwang.local.json` 后，由 `scripts/render_longwang_config.py` 渲染真实配置，再由 `scripts/longwang_doctor.py` 判断哪条 lane 已具备起跑条件。

生成出来的 `reports/`、`runs/`、`targets/`、`memory/` 等目录是运行态结果，不回灌进公开仓。

## 当前一句话结论

`LongWangClaw` 当前应被理解为：从 OpenClaw 控制面与 Super8 Loop9 工作流里摘出的、可回填、可配置、可质检、能承载六段闭环的公开工作区仓库。
