# LongWangClaw｜端到端运行手册

这份手册按真实闭环写：Audit -> Real-PoC 静态编写 -> 静态产物上传 -> 动态环境搭建与 PoC 复现 -> 本地标准交付报告 -> 动态复现后的标准报告上传。

## 0. 装配与配置

先把公开工作区回填到本机 OpenClaw：

```bash
python3 scripts/bootstrap_openclaw_layout.py --workspace-root ~/.openclaw/workspace
```

准备本机私有 profile：

```bash
cp workspace/config/longwang.example.json workspace/config/longwang.local.json
```

填完 `workspace/config/longwang.local.json` 后，先 dry-run：

```bash
python3 scripts/render_longwang_config.py --profile workspace/config/longwang.local.json --dry-run
python3 scripts/longwang_doctor.py --profile workspace/config/longwang.local.json
```

doctor 没有指出配置模板错误后，再写入本机：

```bash
python3 scripts/render_longwang_config.py --profile workspace/config/longwang.local.json --write
```

这一步会写入 OpenClaw / OpenCode / Codex / cron / env 配置，并把 Super8 OpenCode agents 的模型 frontmatter 渲染成 profile 里的本机模型。

## 1. Audit

核心 skill：

```text
~/.openclaw/workspace/skills/loop9-wrapped-audit/SKILL.md
```

本地已有目标源码时，从 Super8 wrapper 发车：

```bash
cd ~/.openclaw/workspace/Super8
./.opencode/_scripts/loop9_authorized_review.py --policy internal-full-audit <local-target-path>
```

微步收录口径或主动发现模式应先读取 `loop9-wrapped-audit` 的 skill 和 references，再物化目标到 `~/.openclaw/workspace/targets/`，最后仍通过同一个 wrapper 发车。

## 2. Real-PoC 静态编写

核心 skill：

```text
~/.openclaw/workspace/skills/loop9-real-poc/SKILL.md
```

入口：

```bash
python3 ~/.openclaw/workspace/skills/loop9-real-poc/scripts/run_loop9_real_poc.py <completed-loop9-run-dir>
```

输出仍留在对应 audit run 下：

```text
<loop9-run-dir>/real_pocs/
  manifest.json
  real_poc_final_status.json
  *.py
  real_poc_solution_vN/
  real_poc_validation_report_vN/
```

## 3. 静态 Audit/PoC 产物上传

核心 skill：

```text
~/.openclaw/workspace/skills/loop9-feishu-publisher/SKILL.md
```

这一段上传的是内部 raw artifacts：Audit 过程、审计结论、PoC 文件、验证材料。它不等于标准交付报告上传。

薄计划入口：

```bash
python3 ~/.openclaw/workspace/skills/loop9-feishu-publisher/scripts/build_publish_plan.py <loop9-run-or-poc-path> --pretty
```

状态文件：

```text
~/.openclaw/workspace/memory/loop9-feishu-publisher-state.json
```

目标 wiki 结构默认是：

```text
Loop9 审计交付中心（内部）
  项目 - <项目名>
    00-项目总览
    01-原始输入
    02-审计过程
    03-审计结论
    04-验证报告
    05-PoC与验证
```

## 4. 动态环境搭建与 PoC 复现

核心 skill：

```text
~/.openclaw/workspace/skills/loop9-verify-v4/SKILL.md
```

子阶段：

- `loop9-verify-v4-env-bootstrap`
- `loop9-verify-v4-finding-replay`
- `loop9-verify-v4-distillation`

自动入口：

```bash
~/.openclaw/workspace/bin/loop9-verify-v4-auto-run.sh
```

该入口会读取渲染后的本机环境文件：

```text
~/.openclaw/workspace/config/longwang.env
```

远程 Docker/CVM、SSH、runtime 根目录、清理保留时间等应由 `longwang.local.json` 统一配置，再由 `render_longwang_config.py` 渲染到 env 文件。没有远程 runtime 配置时，doctor 会把 `dynamic_verify` 标为 blocked。

## 5. 本地标准交付报告

核心 skill：

```text
~/.openclaw/workspace/skills/loop9-delivery-reports/SKILL.md
```

常见入口：

```bash
python3 ~/.openclaw/workspace/skills/loop9-delivery-reports/scripts/build_repo_delivery_reports.py --round-root <reports/.../rounds/<repo>/<round>>
```

或：

```bash
python3 ~/.openclaw/workspace/skills/loop9-delivery-reports/scripts/build_repo_delivery_reports.py --delivery-input-json <delivery-report-input.json>
```

标准输出目录：

```text
~/.openclaw/workspace/reports/<repo>/
  00-索引.md
  01-仓库级中文交付报告.md
  02-仓库级技术汇总.md
  02-仓库级技术汇总.json
  <finding>.md
  poc/
  http/
  98-delivery-bundle.manifest.json
  99-最终本地复盘.md
  99-最终本地复盘.json
```

## 6. 动态复现后的标准报告上传

核心 skill：

```text
~/.openclaw/workspace/skills/loop9-feishu-delivery-publisher/SKILL.md
```

这一段只上传已经生成好的标准报告树，不重新上传 raw audit artifacts。

薄计划入口：

```bash
python3 ~/.openclaw/workspace/skills/loop9-feishu-delivery-publisher/scripts/build_report_publish_plan.py ~/.openclaw/workspace/reports/<repo> --pretty
```

目标 wiki 结构：

```text
项目 - <项目名>
  06-标准交付报告
```

上传前必须确认本地标准报告已经包含：

- `98-delivery-bundle.manifest.json`
- 完成态 `99-最终本地复盘.*`
- PoC 层
- HTTP 证据层
- 必要时的 `03-复现实验信息.md`

## Cron 五条 lane

公开示例：

```text
~/.openclaw/workspace/plans/loop9-hourly-cron-jobs.example.json
```

五条 lane：

- `Loop9 Audit Dispatcher`
- `Loop9 Real-PoC Dispatcher`
- `Loop9 Publish Dispatcher`
- `Loop9 Delivery Report Publisher`
- `Loop9 Verify V4 Auto Runner`

本机真实 cron jobs 应由 `templates/cron/jobs.json.tpl` 从 `longwang.local.json` 渲染，不要把真实 `~/.openclaw/cron/jobs.json` 放进仓库。
