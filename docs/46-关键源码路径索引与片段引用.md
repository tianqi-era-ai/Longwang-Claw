# 龙王小龙虾｜关键源码路径索引与片段引用

这份索引按当前公开仓 `LongWangClaw` 写。它不再把本机 `~/.openclaw` 里的私有路径、运行报告和 target 仓库当成公开仓内容。

## 使用原则

对外引用源码路径时只引用公开仓内的稳定路径：

- `workspace/`
- `extensions/`
- `templates/`
- `scripts/`
- `docs/`
- `harness/`

不要把以下目录当成公开仓内容引用：

- `reports/`
- `runs/`
- `targets/`
- `memory/`
- `logs/`
- 真实 `openclaw.json`
- 真实 `opencode.json`
- 真实 Codex `config.toml`

这些只能作为用户本机运行后生成或填写的私有运行态。

## 运行态输出怎么引用

运行后可能出现：

- `~/.openclaw/workspace/Super8/temp/loop9/...`
- `~/.openclaw/workspace/Super8/temp/loop9-observe/...`
- `~/.openclaw/workspace/runs/...`
- `~/.openclaw/workspace/reports/<repo>/...`
- `~/.openclaw/workspace/targets/<repo>/...`
- `~/.openclaw/workspace/memory/...`

这些可以在用户本机排障或宣发案例整理时引用，但不要当成公开仓内容提交。

标准交付报告树的预期形态见：

- `docs/端到端运行手册.md`
- `workspace/skills/loop9-delivery-reports/SKILL.md`
- `workspace/skills/loop9-feishu-delivery-publisher/references/layout.md`

## 不建议怎么引用

- 不要把 `reports/<repo>/...` 写成仓库内置样例。
- 不要把 `targets/<repo>/...` 写成公开仓必须携带的 demo。
- 不要把 `loop9_env_poc_dynamic_verify`、旧 `loop9-env-bootstrap` 或旧 `loop9-poc-verification` 写成当前 active 主入口。
- 不要把 `extensions/openclaw-lark/` 的 token 存储、OAuth 字段误读成公开密钥；真实授权态由用户本机管理。
- 不要把某个 skill 文件讲成完整产品全貌；skill 是任务组织层，Dispatcher 和运行态状态面仍然重要。
- 不要把宣发/运营材料或首发案例名单当成当前公开仓资产；这些只留给宣发侧人工选材。

## 当前一句话总结

当前最值得引用的路径锚点是：装配配置质检、OpenClaw 控制面、Super8 Loop9 主线、六段 skill 闭包、cron 长任务推进、Harness 高级控制面。运行报告、target 源码、账号态配置、宣发材料、首发案例名单和缓存日志都不属于当前公开仓内容。
