# 语言正则使用边界说明

## 当前原则
- 已验证的长正则资产，优先**原样保留**，不要为了“更优雅”去重构它。
- 语言正则是**辅助检索层**，不是最终漏洞判断层。
- 大模型负责：
  - 判断当前项目是否属于对应语言生态
  - 决定何时调用脚本
  - 决定如何解释命中结果

## 当前已实现语言
- `java`
- `php`

## 什么时候调用 Java 正则脚本
当大模型已经判断当前项目明显属于 Java / JVM 生态时，再调用：
- 常见信号：`.java`、`pom.xml`、`build.gradle`、Spring / Servlet / Shiro / Fastjson / Jackson / XStream / Quartz 等上下文
- 当前建议调用方式：
  - `python3 scripts/rce_regex_scan.py <repo-root> --language java`
- 默认行为：
  - 扫完整个候选文件集合（不再提供 `--limit-files` 给正常调用路径）
  - 把完整结果保存到 repo 内的结果文件
  - stdout 只输出简短 summary（避免上下文膨胀）

## 什么时候调用 PHP 正则脚本
当大模型已经判断当前项目明显属于 PHP 生态时，再调用：
- 常见信号：`.php`、`composer.json`、`composer.lock`、`.phtml`、`.inc`、`Twig`、`Smarty`、`Blade`、`Phar`、`unserialize`、`move_uploaded_file` 等上下文
- 当前建议调用方式：
  - `python3 scripts/rce_regex_scan.py <repo-root> --language php`
- 默认行为：
  - 扫完整个候选文件集合
  - 把完整结果保存到 repo 内的结果文件
  - stdout 只输出简短 summary（避免上下文膨胀）

## 为什么不能直接把命中当成结论
因为这条长正则追求的是高召回，不是高精度。
例如：
- `jackson`
- `gson`
- `readValue`
- `exec`
- `Process`

单独命中它们，并不等于漏洞成立。

## 当前 Java 文件范围策略
当前脚本会优先扫描：
- 源码/模板相关：`.java`, `.jsp`, `.jspx`, `.tag`, `.tagx`, `.tld`
- 配置相关：`.xml`, `.yml`, `.yaml`, `.properties`
- 构建/脚本相关：`.gradle`, `.groovy`, `.kt`, `.kts`
- 重点文件名：`pom.xml`, `build.gradle`, `build.gradle.kts`, `settings.gradle`, `settings.gradle.kts`, `gradle.properties`, `web.xml`, `beans.xml`, `application.properties`, `application.yml`, `application.yaml`

## 当前 PHP 文件范围策略
当前脚本会优先扫描：
- 源码/模板相关：`.php`, `.phtml`, `.php3`, `.php4`, `.php5`, `.php7`, `.php8`, `.phps`, `.inc`, `.module`, `.install`, `.theme`, `.tpl`, `.twig`, `.latte`, `.ctp`, `.phar`, `.stub`
- 配置相关：`.yml`, `.yaml`, `.ini`
- 重点文件名：`composer.json`, `.htaccess`, `php.ini`, `config.php`, `index.php`

## 当前默认不优先扫描的目录
- `.git`, `.idea`, `.vscode`, `.gradle`
- `node_modules`, `vendor`
- `dist`, `build`, `target`, `out`
- `coverage`, `__pycache__`
- `.mvn`, `.svn`, `.hg`
- `tmp`, `temp`, `logs`, `cache`, `storage`

## 未来扩展方式
脚本入口已经按 `--language` 设计，可继续扩展：
- `php`
- `python`
- `node`
- `dotnet`

但原则不变：
- 每种语言优先使用单独、成熟、经过检验的资产
- 不要为了统一而硬拼成一个跨语言 giant regex
