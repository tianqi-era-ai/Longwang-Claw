# 调用策略与判定原则

> 这个文件不是主说明书，而是给使用该 skill 的执行者看的判定参考。

---

# 1. 调用框架：按阶段，不按死次数

默认放在两个阶段使用：

1. **早期轻量预扫阶段**
   - repo 已经本地化
   - 主审计已做初步结构理解
   - 尚未完全锁死到单个漏洞方向

2. **强信号深挖阶段**
   - 审计过程中出现上传、模板、preview、plugin、JWT/SSO、弱鉴权、低权限后台、暴露管理面等强信号

注意：
- 这里约束的是 **阶段框架**，不是每个阶段只能调用一次。
- 在强信号阶段，如果连续出现多个明确线索，允许多次调用。
- 是否继续投入，以及投入多少时间/token，由执行者根据线索质量自行判断。

---

# 2. 什么叫“强信号”

以下都可视为强信号：

## 2.1 上传相关
- 可上传图片/附件/模板/主题/插件
- 可远程抓取文件或导入文件
- 上传落点可预测
- 扩展名/MIME/内容校验出现分离或不一致

## 2.2 模板/渲染相关
- theme/template/email-template/page-builder/design-xml
- preview/render/compile/evaluate/expression
- 用户输入进入模板语境

## 2.3 配置/集成/执行扩展相关
- webhook / callback / test connection
- scheduler / hook / script runner
- plugin / package / extension / installer
- JWT / SSO / auth provider / user-source 配置

## 2.4 鉴权边界相关
- 自注册后能触达高价值功能面
- guest/editor/support/marketing 等低权限角色接近模板/配置/上传/预览/执行能力
- 疑似越权、权限模型异常、后台 API 弱保护

## 2.5 中间件/控制台相关
- 暴露管理面
- 可改配置项可能开启模板/脚本/动态执行能力

---

# 3. 什么时候不值得立刻调用

以下情况更适合先记一笔，再看后续是否升温：
- 只有非常浅的危险关键词命中
- 只有模糊目录名，没有功能证据
- 只有普通上传功能，但暂时看不到任何执行/渲染/路径控制线索
- 只有普通后台配置，但看不到它与执行面的连接

原则：
- 不要把 skill 的调用写成机械条件触发器
- 但也不要因为“怕打扰”而错过真正强信号

---

# 4. 这类 skill 的正确产物

一次好的调用，通常应该带来：
- 高价值功能面排序
- 1~3 条成链假设
- 对微步偏好的贴合度判断
- 对 `RCE专题共享上下文.md` 的增量更新
- 明确建议：继续深挖 / 保留观察 / 暂时降级

---

# 5. 这类 skill 的常见失败方式

- 把它当危险函数 grep 包装器
- 把所有上传点都抬成高危
- 看到模板/渲染就直接判 RCE
- 因为“发现了一个点”就强迫自己必须调用一次
- 因为“已经调用过了”就拒绝在强信号阶段再次调用

---

# 6. 当前推荐顺序

当前正确顺序应是：
1. 先把该能力打磨成独立可插拔 skill
2. 再考虑如何嵌入现有 Loop9 audit 主提示词
3. 再按真实使用效果决定是否扩展并列 skill（如 文件上传 / 越权 等）
