# Telegram / 小龙虾 通知模板

> 目的：统一长任务在聊天侧的通知口径，减少重复解释，避免遗漏关键信息。

---

## 1. 启动成功通知

可用模板：

```text
已启动 Super8 的标准化 Loop9 长任务。

目标：<target-path>
入口：wrapper（不是手工拼命令）
模式：<agent / command>
特性：后台长跑，不会因为短时间无输出就判定失败。

我会在有实质进展或最终结果时再通知你。
```

---

## 2. 中途进展通知

适用时机：
- 已开始 repo 结构探索
- 已生成共享上下文
- 已出现初步报告文件
- 已确认关键高危结果

可用模板：

```text
Super8 / Loop9 长任务有新进展：

- 当前阶段：<阶段名>
- 目标：<target-path>
- 新产物：<file-paths>
- 简述：<1~3 条关键进展>

任务仍在后台继续运行，我会在有下一次实质变化时再同步。
```

---

## 3. 最终完成通知

可用模板：

```text
Super8 / Loop9 长任务已产出可用结果。

目标：<target-path>
主要结果文件：
- <report-1>
- <report-2>
- <shared-context>

摘要：
- <关键结论 1>
- <关键结论 2>
- <关键结论 3>

如果你要，我下一步可以继续：
1. 给你整理成人类可读摘要
2. 生成修复优先级建议
3. 继续做后续封装/自动化
```

---

## 4. 卡住/停滞通知

适用时机：
- 长时间没有新文件增长
- session 停在同一状态
- 明显异常但还没彻底失败

可用模板：

```text
Super8 / Loop9 长任务目前疑似停滞。

目标：<target-path>
已观察时长：<duration>
当前表现：<现象>
最近产物：<file-paths>

我暂时不会主动 kill。除非你要求我停掉，或者后续确认它已经异常退出。
```

---

## 5. 异常退出通知

可用模板：

```text
Super8 / Loop9 长任务已异常退出。

目标：<target-path>
最后阶段：<阶段>
可用产物：<file-paths>
异常线索：<stderr/session/log 关键信息>

我可以下一步：
1. 帮你归纳失败原因
2. 修正入口/提示词/控制器后重跑
3. 提取已有中间产物，避免完全重来
```
